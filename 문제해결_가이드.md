# 다른 컴퓨터에서 작동하지 않을 때 문제 해결 가이드

## 🔍 주요 원인 및 해결 방법

### 1️⃣ **파일을 직접 더블클릭해서 열었을 때 (file:// 프로토콜)**

#### 문제 원인:
- **로컬 파일 시스템 제한**: `file://` 프로토콜로 HTML 파일을 열면 브라우저의 보안 정책으로 인해 많은 기능이 제한됩니다.
- JavaScript의 일부 기능(Geolocation API, localStorage 등)이 제대로 작동하지 않을 수 있습니다.

#### 해결 방법:

**방법 A: 웹 서버 사용 (권장)**
```batch
# start.bat 파일 실행
start.bat

# 또는 start_tablet.bat 실행 (네트워크 접근 가능)
start_tablet.bat
```

**방법 B: 간단한 Python 서버**
```bash
# Python이 설치되어 있는 경우
python -m http.server 8000

# 그 다음 브라우저에서
http://localhost:8000/standalone.html
```

**방법 C: GitHub Pages 배포**
- GitHub에 업로드하여 인터넷을 통해 접근
- 가장 안정적이고 접근성이 좋은 방법

---

### 2️⃣ **파일 인코딩 문제**

#### 문제 원인:
- Windows에서 다른 인코딩으로 저장되었거나 복사 과정에서 인코딩이 깨진 경우
- 한글이 깨져서 표시될 수 있습니다.

#### 해결 방법:
1. 메모장 또는 VS Code로 `standalone.html` 파일 열기
2. **파일 → 다른 이름으로 저장**
3. **인코딩**: `UTF-8` 선택
4. 저장

또는 VS Code 사용 시:
- 우측 하단의 인코딩 표시 클릭
- "인코딩하여 저장" → "UTF-8" 선택

---

### 3️⃣ **브라우저 보안 정책 (Geolocation API)**

#### 문제 원인:
- **위치 권한**: 다른 컴퓨터에서 처음 접속하면 위치 권한을 새로 요청합니다.
- **HTTPS 필요**: 일부 브라우저는 Geolocation API에 대해 HTTPS만 허용합니다 (localhost는 예외).

#### 해결 방법:
1. 브라우저에서 위치 권한을 **허용**하도록 설정
2. `http://localhost`로 접속 (file://가 아닌 HTTP 프로토콜 사용)
3. 브라우저 설정에서 위치 권한 확인:
   - Chrome: 주소창 왼쪽 자물쇠 아이콘 → 위치 → 허용
   - Edge: 주소창 왼쪽 자물쇠 아이콘 → 위치 → 허용

---

### 4️⃣ **로컬 스토리지 (localStorage) 문제**

#### 문제 원인:
- 브라우저 설정에서 쿠키/로컬 스토리지가 비활성화된 경우
- 시크릿 모드/프라이빗 브라우징 모드 사용 시

#### 해결 방법:
1. 일반 브라우징 모드 사용 (시크릿 모드 아님)
2. 브라우저 설정에서 쿠키 및 사이트 데이터 허용 확인
3. 브라우저 캐시 삭제 후 재시도

---

### 5️⃣ **파일 경로 문제**

#### 문제 원인:
- 파일이 다른 폴더나 드라이브로 이동된 경우
- 상대 경로를 사용하는 외부 리소스가 있다면 경로 문제 발생 가능

#### 해결 방법:
- `standalone.html`은 모든 리소스가 내장되어 있어야 하므로, 단일 파일만 복사하면 됩니다.
- **반드시 `standalone.html` 파일 하나만 다른 컴퓨터로 복사하면 됩니다.**

---

### 6️⃣ **브라우저 호환성**

#### 문제 원인:
- 오래된 브라우저 사용
- Internet Explorer 사용 (현재 지원되지 않음)

#### 해결 방법:
- **최신 버전의 Chrome, Edge, Firefox, Safari 사용**
- Internet Explorer는 사용하지 마세요 (2022년 지원 종료)

---

### 7️⃣ **JavaScript 오류**

#### 문제 원인:
- 브라우저의 JavaScript가 비활성화된 경우
- 브라우저 확장 프로그램이 JavaScript 실행을 막는 경우

#### 해결 방법:
1. 브라우저 개발자 도구 열기 (F12)
2. Console 탭에서 오류 메시지 확인
3. JavaScript가 활성화되어 있는지 확인:
   - Chrome/Edge: 설정 → 사이트 설정 → JavaScript → 허용
4. 확장 프로그램 비활성화 후 재시도

---

### 8️⃣ **방화벽/보안 소프트웨어**

#### 문제 원인:
- Windows 방화벽이 로컬 서버 연결을 차단
- 안티바이러스 소프트웨어가 차단

#### 해결 방법:
1. Windows 방화벽에서 Python/PowerShell HTTP 서버 허용
2. 안티바이러스 예외 목록에 프로젝트 폴더 추가

---

## ✅ 체크리스트 (문제 해결 순서)

다른 컴퓨터에서 실행하기 전:

- [ ] `standalone.html` 파일 하나만 복사했는지 확인
- [ ] 파일 인코딩이 UTF-8인지 확인
- [ ] 최신 브라우저(Chrome/Edge/Firefox) 사용 중인지 확인
- [ ] `file://` 프로토콜이 아닌 HTTP 서버로 실행하는지 확인
- [ ] 브라우저 개발자 도구(F12)에서 오류 메시지 확인
- [ ] 위치 권한이 허용되어 있는지 확인
- [ ] 시크릿 모드가 아닌 일반 브라우징 모드 사용

## 🚀 가장 확실한 해결 방법

### GitHub Pages에 배포하기 (권장)

1. GitHub에 `standalone.html` 파일을 `index.html`로 이름 변경하여 업로드
2. GitHub Pages 활성화
3. 인터넷이 연결된 모든 기기에서 접근 가능
4. 인코딩, 경로, 프로토콜 문제 모두 해결됨

**자세한 방법**: `GITHUB_배포_가이드.md` 참고

## 📞 문제가 계속되면 확인할 것

1. **브라우저 개발자 도구(F12) → Console 탭**에서 오류 메시지 확인
2. **Network 탭**에서 리소스 로딩 실패 여부 확인
3. 어떤 기능이 작동하지 않는지 구체적으로 기록:
   - 지도가 안 보임
   - 로그인이 안 됨
   - 버튼 클릭이 안 됨
   - 등등...
